Module: cam_control¶

Camera device abstraction layer.

Wrapper for Basler’s pylon software library, which is based on the GenICam standard.

See also

GenICam Standard

Standard developed for a unified programming interface for cameras. The Basler camera we use is compliant with GenICam.

Modules | Basler Product Documentation

C++ API documentation for Basler cameras (modules Instant Camera and Universal Instant Camera are of interest).

Parameters | Basler Product Documentation

List of parameters acceptable for camera objects.

basler/pypylon: The official python wrapper for the pylon Camera Software Suite

Python wrapper used in this project for pylon.

class PyQSense.cam_control.Camera_Event_Handler(*args: Any, **kwargs: Any)

Custom camera event handling.

Wraps the pylon API’s camera event handler Pylon::CCameraEventHandler, while providing two additional attributes.

running

Flag set to indicate the camera is active.

Type:

bool

stop_flag

Flag to stop camera in the case of an error event.

Type:

bool

__init__()

Wrapper for Pylon::CCameraEventHandler init.

It also initializes running and stop_flag attributes to False.

OnCameraEvent(camera, user_provided_id, node)

Set stop flag if an overtrigger is detected.

Parameters:

  • camera (pypylon.pylon.InstantCamera) – The camera on which the event occurred.

  • user_provided_id (int) – Used to distinguish between camera events. In the C++ API, this is really of type intptr_t.

  • node (pypylon.genicam.INode) – The GenICam node from which the event is originating.

class PyQSense.cam_control.Basler_Camera(frame_trigger, burst_trigger)

Control a Basler camera.

camera

Python-wrapped instance of pylon C++ API’s Pylon::CInstantCamera class.

Type:

pypylon.pylon.InstantCamera

event_handler

Instance of Camera_Event_Handler class.

Type:

pypylon.pylon.CameraEventHandler

__init__(frame_trigger, burst_trigger)

Initialize the Basler camera.

Parameters:

  • frame_trigger (str) – The I/O line which triggers a single image acquisition. For example, “Line1”.

  • burst_trigger (str) – The I/O line which triggers a acquisition of a burst of images.

  • args (dict) – Additional camera properties to be set. These are GenICam properties.

set_params(**args)

Set additional parameters of the camera.

Camera parameters are GenICam nodes; for this method to work, they must support the GetValue() method.

The Pylon::CBaslerUniversalInstantCamera class automatically creates members for each node in the camera description file; their values can be accessed through class member access. Each of these members is a node type, so they have the methods specified in the GenICam standard, for example GetValue(). Refer to the GenICam standard and the Basler camera product documentation for additional information.

Parameters:

args (dict) – Keyword arguments representing the camera properties and their corresponding values.

get_param(node_name)

Retrieves the value of a parameter from the camera.

Parameters:

node_name (str) – The name of the parameter to retrieve.

Returns:

The value of the parameter. The return type depends on the type of parameter.

run_acquisition(n_samples, sample_frames, timeout=1000)

Retrieves a sequence of images from the camera.

Parameters:

  • n_samples (int) – Number of image samples (collections of multiple images) to take.

  • sample_frames (int) – Number of raw images per sample.

  • timeout (int) – Timeout value for retrieving an image capture in milliseconds. Defaults to 1000.

Returns:

Array of images with shape (n_samples, sample_frames, Height, Width), where Height and Width are camera parameters (GenICam nodes).

Return type:

numpy.ndarray

Raises:

RuntimeError – A camera overtrigger event has been detected.

open()

Opens the camera device.

close()

Closes the camera device.
