ttl_control#

Manage PulseBlaster TTL pulse generators.

Abstraction layer over SpinAPI, the official C library for PulseBlaster cards. Implements a PyQSense.ttl_control.Pulse_Blaster class with methods for starting, stopping, and programming a repeating binary signal to the card.

Some vocabulary:

Instruction word

An 80-bit PulseBlaster processor instruction.

Instruction

Human-readable name given to the opcode field of an instruction word. For example, opcode 6 is named “BRANCH”.

Flags

A 24-bit integer which represents the state of the PulseBlaster output lines. A mapping of each bit to the corresponding output pin number is found in table 2 of Using SpinAPI in C/C++ for PulseBlaster Programming.

class PyQSense.ttl_control.ttl_channel(name, channel_number, start_offset, width_offset)

TTL channel configuration object.

Represents a physical output line (channel) on the TTL card. Offsets to the pulse timings and duration may be specified here. This class is not defined explicitly; rather, it is instantiated by the standard library collections.namedtuple factory function. Stores offset values that are to be used in Pulse_Blaster._event_scheduler. Note that the yaml configuration parser expects start_offset and width_offset to be provided in seconds, while internally, the PulseBlaster expects nanoseconds.

program(channel_sequences)

Program the Pulse Blaster with a new pulse sequence.

Upload a specific pulse sequence to the PulseBlaster. At the core of this method is the pb_inst_pbonly C function which uploads an instruction word to the PulseBlaster’s internal memory. The channels parameter is converted into a series of PulseBlaster instructions corresponding with what is specified in each ttl_channel of channels. The resulting set of instruction words is a simple loop, composed of CONTINUE instructions until the last instruction word, which is a BRANCH to the first instruction word. The flags on each instruction word are specified by the start times and duration times in each of the ttl_channel instances.

Parameters:

channel_sequences (list[ttl_channel]) – List of channel pulse sequence configurations.
